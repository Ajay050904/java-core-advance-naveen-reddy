1-> What is Abstract Data Type (ADT)?
    . Definition:
                1-> Abstract Data Type (ADT) is a data type that defines data and the operations on that data without specifying the implementation details.
                2-> ADT is a logical description of a data structure that defines data and operations but not the implementation.
    . A conceptual model
    . Defines what data is stored
    . Defines what operations can be performed
    . Does N  OT specify how it is implemented
    . ðŸ“Œ Implementation may differ from language to language, but the concept remains the same.
    . Examples of ADT:
                    Array
                    List
                    Stack
                    Queue
                    Set
                    Map
    . Components of ADT:
                    1-> Data
                    2-> Operations on the data
    
2-> Why is it called â€œabstractâ€?
    . Because it hides implementation details and focuses only on behavior and operations.

3-> What is the difference between data type and ADT?
    . A data type specifies how data is stored, while ADT specifies what operations can be performed.

4-> Does ADT depend on programming language?
    . No, the concept remains the same; only implementation changes.

5-> Why is ADT important?
    . It helps in designing efficient programs and choosing the right data structure.
    . It separates "logic from implementation", making programs easier to design and maintain

6-> Is stack an ADT or data structure?
    . Stack is an "ADT"; its implementation using array or linked list is a "data structure".

7-> Can one ADT have multiple data structures?
    . Yes, For example, Stack can be implemented using array or linked list.
